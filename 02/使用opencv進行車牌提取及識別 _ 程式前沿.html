車牌提取過程

一個典型的車輛牌照識別系統一般包括以下4個部分：車輛影象獲取、車牌定位、車牌字元分割和車牌字元識別 。

本篇文章只介紹車牌的提取分割過程，關於如何對分割後的字元進行識別 ，請看我的另一篇WIKI ，opencv基於人工神經網路的字母識別。

車輛影象獲取

車輛影象獲取是車牌識別的第一步，也是很重要的一步，車輛影象的好壞對後面的工作有很大的影響。如果車輛影象的質量太差，連人眼都沒法分辨，那麼肯定不會被機器所識別出來。車輛影象都是在實際現場拍攝出來的，實際環境情況比較複雜，影象受天氣和光線等環境影響較大，在惡劣的工作條件下系統效能將顯著下降。 現有的車輛影象獲取方式主要有兩種：一種是由彩色攝像機和影象採集卡組成，其工作過程是：當車輛檢測器（如地感線圈、紅外線等）檢測到車輛進入拍攝範圍時，向主機傳送啟動訊號，主機通過採集卡採集一幅車輛影象，為了提高系統對天氣、環境、光線等的適應性，攝像機一般採用自動對焦和自動光圈的一體化機，同時光照不足時還可以自動補光照明，保證拍攝圖片的質量；另一種是由數碼照相機構成，其工作過程是：當車輛檢測器檢測到車輛進入拍攝範圍時，直接給數碼照相機傳送一個訊號，數碼相機自動拍攝一幅車輛影象，再傳到主機上，數碼相機的一些技術引數可以通過與數碼相機相連的主機進行設定，光照不足時也需要自動開啟補光照明，保證拍攝圖片的質量。

車牌定位

車牌定位的主要工作是從攝入的汽車影象中找到汽車牌照所在位置，並把車牌從該區域中準確地分割出來，供字元分割使用。因此，牌照區域的確定是影響系統效能的重要因素之一，牌照的定位與否直接影響到字元分割和字元識別的準確率。目前車牌定位的方法很多，但總的來說可以分為以下4類：（1）基於顏色的分割方法，這種方法主要利用顏色空間的資訊，實現車牌分割，包括彩色邊緣演算法、顏色距離和相似度演算法等；（2）基於紋理的分割方法，這種方法主要利用車牌區域水平方向的紋理特徵進行分割，包括小波紋理、水平梯度差分紋理等；（3）基於邊緣檢測的分割方法；（4）基於數學形態法的分割方法。 本文為了程式碼實現上的方便，我採用的是基於邊緣檢測的分割方法。主要是利用水平投影方法和垂直投影方法進行車牌定位。

車牌字元分割

要識別車牌字元，前提是先進行車牌字元的正確分割與提取。字元分割的任務是把多列或多行字元影象中的每個字元從整個影象中切割出來成為單個字元。車牌字元的正確分割對字元的識別是很關鍵的。傳統的字元分割演算法可以歸納為以下三類：直接分割法、基於識別基礎上的分割法、自適應分割線類聚法。直接分割法簡單，但它的侷限是分割點的確定需要較高的準確性；基於識別基礎上的分割法是把識別和分割結合起來，但是需要識別的高準確性，它根據分類和識別的耦合程度又有不同的劃分；自適應分割線聚類法是要建立一個分類器，用它來判斷影象的每一列是否是分割線，它是根據訓練樣本來進行自適應學習的神經網路分類器，但對於粘連字元訓練困難。也有直接把字元組成的單詞當作一個整體來識別的，諸如運用馬爾科夫數學模型等方法進行處理，這些演算法主要應用於印刷體文字識別。

Candy邊緣檢測

Canny邊緣檢測運算元的方向性質保證了很好的邊緣強度估計，而且能同時產生邊緣梯度方向和強度兩個資訊，即能在一定程度上抗噪聲又能保持弱邊緣，因此採用以canny運算元做邊緣檢測。

Canny演算法步驟：

(1)去噪

任何邊緣檢測演算法都不可能在未經處理的原始資料上很好地處理，所以第一步是對原始資料與高斯 mask 作卷積，得到的影象與原始影象相比有些輕微的模糊（blurred）。這樣，單獨的一個畫素雜訊在經過高斯平滑的影象上變得幾乎沒有影響。

(2)用一階偏導的有限差分來計算梯度的幅值和方向。

(3)對梯度幅值進行非極大值抑制。

車牌細定位的目的是為下一步字元的分割做，就是要進一步去掉車牌冗餘的部分。在一幅經過適當二值化處理 含有車牌的影象中，車牌區域具有以下三個基本特徵：

1.在一個不大的區域內密集包含有多個字元；

2.車牌字元與車牌底色形成強烈對比；

3.車牌區域大小相對固定，區域長度和寬度成固定比例。

形態學（膨脹腐蝕）處理

膨脹與腐蝕的處理效果就如其名字一樣，我們通過膨脹連線相近的影象區域，通過腐蝕去除孤立細小的色塊。通過這一步，我們希望將所有的車牌號字元連通起來，這樣為我們接下來通過輪廓識別來選取車牌區域做準備。由於字元都是橫向排列的，因此要連通這些字元我們只需進行橫向的膨脹即可。

進行膨脹腐蝕操作需要注意的是要一次到位，如果一次膨脹沒有連通到位，那麼再次腐蝕將會將影象回覆原裝，因此我首先做了2次迭代的膨脹，保證數字區域能連通起來，再進行4次迭代腐蝕，儘可能多的去除小塊碎片，隨後2次迭代膨脹，保證膨脹次數與腐蝕次數相同，以回覆連通區域形態大小。

矩形輪廓查詢與篩選：經過上一步操作，理論上來說車牌上的字元連通成一個矩形區域，通過輪廓查詢我們可以定位該區域。當然，更為準確的說，經過上面的操作，我們將原始圖片中在X方向排列緊密的縱向邊緣區域連通成了一個矩形區域，出了車牌符合這個特點外，其他一些部分如路間欄杆，車頭的紋理等同樣符合。因此我們會找到很多這樣的區域，這就需要我們進一步根據一些關於車牌特點的先驗知識對這些矩形進行進一步篩選。最終，定位車牌所在的矩形區。

畫素值判斷

這裡主要使用水平對映和垂直對映畫素進行畫素級分割。

分割方法：首先判斷每一列的畫素值大於0的畫素個數超過5個時，認為此列是有數字的，記錄每列畫素是否大於5，產生一個陣列,陣列的值有可能是一下。

a[100] = {00000001001110011111111111111100001111111111000000011111111100000000000001111110000110101011110000000000000000。。。。。。。}

這裡做一個簡單的水平方向的濾波，Size大小是5 ，然後對 每一個畫素做處理，處理方法：

        //對陣列進行濾波，減少突變概率
for (int i = 2; i < roi_col - 1 - 2; i  ) {
if ((pixrow[i - 1]   pixrow[i - 2]   pixrow[i   1]   pixrow[i   2]) >= 3) {
pixrow[i] = 1;
}
else if((pixrow[i - 1]   pixrow[i - 2]   pixrow[i   1]   pixrow[i   2]) <= 1) {
pixrow[i] = 0;
}
}

如果 該畫素周圍為1的畫素大於等於3個，不算自己 那麼久認為該畫素為 1， 如果 該畫素周圍 畫素為0的個數小於等於1 ，那麼就認為該畫素為0

float sumValue = 0;

c = r*c;
r = 1;
}
for (int i = 0; i < r; i ) {

}

void calcGradientFeat(Mat & imgSrc, vector<float> & feat) {
Mat image;
cvtColor(imgSrc, image, CV_BGR2GRAY);
resize(image, image, Size(8, 16));

Mat y_mask = Mat(3, 3, CV_32F, mask) / 8;

Mat sobelX, sobelY;

filter2D(image, sobelX, CV_32F, x_mask);
filter2D(image, sobelY, CV_32F, y_mask);
sobelX = abs(sobelX);
sobelY = abs(sobelY);

float totleValueX = sumMatValue(sobelX);
float totleValueY = sumMatValue(sobelY);
